# Code of Conduct

## Our Standard

This project aims to be practical, respectful, and technically rigorous.

Examples of behavior that contribute to a positive environment:

- giving actionable technical feedback
- assuming good intent while discussing tradeoffs directly
- documenting concerns clearly and concretely
- keeping criticism focused on code, behavior, and process

Examples of unacceptable behavior:

- harassment, abuse, or intimidation
- discriminatory language or personal attacks
- doxxing or publishing private information
- deliberately disruptive behavior in issues, pull requests, or discussions

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct.

## Reporting

If you experience or witness unacceptable behavior, use any private contact method published on the maintainer's GitHub profile. If no private channel is available, open a minimal public issue requesting direct contact without posting sensitive details.

## Scope

This Code of Conduct applies within project spaces and in public spaces when someone is representing the project.
