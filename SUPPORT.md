# Support

## When To Open An Issue

Open a GitHub issue for:

- reproducible bugs
- incorrect or misleading documentation
- gaps in the built-in workflows or templates
- requests for improvements that fit the repository's goals

When possible, include the exact command, workflow, PRD shape, and observed output.

## Before Asking For Help

Check:

- `README.md`
- `CONTRIBUTING.md`
- `docs/architecture.md`
- the relevant workflow or script under `.github/` or `scripts/ralph/`

## What This Repository Supports

This repository supports:

- structured PRD-driven planning
- Claude Code based interactive execution
- GitHub Actions automation for PRD creation and Ralph Loop execution

This repository does not provide:

- a standalone product runtime
- support for every editor or CI system
- guarantees that every downstream repository can adopt the template without customization

## Security Issues

Do not report vulnerabilities in public issues. Use `SECURITY.md` instead.
