# Ubiquitous Language Dictionary

This directory maintains the project's ubiquitous language — shared terms and definitions that the entire team uses consistently.

## Format

Each term entry follows this structure:

| Column | Description |
|--------|-------------|
| Term | The canonical name used across the project |
| Definition | A concise, unambiguous explanation |
| Context | Where or when this term is typically used |
| Aliases | Alternative names that should be mapped to this term |

## When to Update

- When a new domain term emerges during PRD or specification creation
- When an existing term is redefined or clarified
- When ambiguity is discovered in team communication or documentation
- During `/docs-review` when terminology inconsistency is found
